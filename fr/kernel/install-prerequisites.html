Obtenir les sources du Kernel ZOO¶

Utilisez la commande suivante pour obtenir le code source du Kernel ZOO via Subversion:

svn checkout http://svn.zoo-project.org/svn/trunk zoo-project

Pour les utilisateurs qui ont un compte développeur, utilisez ce qui suit:

sed "s:\[tunnels\]:\[tunnels\]\nzoosvn = /usr/bin/ssh -p 1046:g" -i ~/.subversion/config
svn co svn+zoosvn://svn.zoo-project.org/var/svn/repos/trunk zoo-project

La première line d’instruction ci-dessus définit un tunnel spécifique pour accéder au serveur svn via le protocole SSH. En effet, le serveur SVN ZOO écoute sur le port 1046 (1024+22) plutôt que sur celui par défaut (22).

Prérequis¶

Les bibliothèques suivantes sont requises sur votre système avant que pouviez installer le Kernel ZOO:

  • autoconf ( http://www.gnu.org/software/autoconf/ )
  • cgic ( http://www.boutell.com/cgic )
  • cURL ( http://curl.haxx.se )
  • FastCGI ( http://www.fastcgi.com )
  • Flex & Bison ( http://flex.sourceforge.net/ http://www.gnu.org/software/bison/ )
  • libxml2 ( http://xmlsoft.org )
  • OpenSSL ( http://www.openssl.org )
  • Python ( http://www.python.org )

Les bibliothèques optionnelles recouvrent:

  • MapServer (optionnel pour les sorties WMS, WFS et WCS) ( http://mapserver.org )

  • PHP encapsulé (optionnel) ( http://www.php.net )

  • SDK Java (optionnel) ( http://java.sun.com )

  • SpiderMonkey (optionnel) ( http://www.mozilla.org/js/spidermonkey/ )

Compiler libcgic¶

La première étape est de compiler libcgic depuis le répertoire zoo-project/thirds. Pour cette tâche, merci d’utiliser la commande suivante:

cd thirds/cgic206
make

Assurez-vous que libcgic.a est créé dans votre répertoire zoo-project/thirds/cgic206. Si oui, alors vous pouvez passer à l’étape suivante.

Sous Windows, plutôt que d’utiliser la commande make, merci d’utiliser:

nmake /f makefile.vc.

Warning

Si vous ne compilez pas libcgic en premier et essayez de compiler le Kernel ZOO, vous aurez une erreur du type cannot find -lcgic
